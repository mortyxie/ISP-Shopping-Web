# ISP-Shopping-Web

二手唱片购物平台 - "凤凰花开的路口"

## 项目概述

本项目是一个二手唱片购物平台，采用前后端同仓库、同部署的方式，仅通过目录进行前后端区分。

- **前端**：Vue.js + Vite
- **后端**：SQLite（待开发）
- **整体色调**：红色调（凤凰花树的颜色）

## 目录结构约定

```
ISP-Shopping-Web/
  frontend/          # Vue 前端应用
    src/             # 前端源码（组件、页面、路由、状态管理等）
    public/          # 静态资源与入口模板
  backend/           # 后端服务（SQLite）
    src/             # 后端源码（路由、中间件、业务逻辑）
    db/              # SQLite 数据库文件与初始化脚本
  shared/            # 前后端共享的类型、常量、工具
  configs/           # 运行配置与环境示例
  doc/               # 需求、计划、设计等文档
```

## 放置规则

- `frontend/`：只放前端相关内容，不放后端代码。
- `backend/`：只放后端相关内容，不放前端代码。
- `shared/`：前后端都要用的类型/常量/工具放这里，避免重复。
- `configs/`：放配置模板（如 `.env.example`），敏感信息不要提交。
- `doc/`：项目说明、设计文档、会议纪要等。

## 重要说明

### ⚠️ 假数据使用

**当前开发阶段，前端使用大量假数据进行开发测试**：

- **用户认证**：使用假账户数据（`src/utils/auth.js`）
- **购物车**：使用 localStorage 存储（`src/utils/cart.js`）
- **商品数据**：主页、商品详情、分类等页面使用硬编码的假数据
- **论坛数据**：论坛页面使用假数据

**所有假数据位置和替换方案已详细记录在 [`frontend/README.md`](./frontend/README.md) 的"假数据使用清单"章节中。**

待后端API开发完成后，需要按照清单逐步替换为真实接口调用。

## 协作说明

1. 新增文件前先确认所属目录，保证边界清晰。
2. 任何跨端共享内容优先放 `shared/`，并在代码中引用。
3. 如果需要新增新的一级目录，请先在 `README.md` 中补充说明。
4. **使用假数据时，必须在 `frontend/README.md` 的"假数据使用清单"中记录**，方便后续API替换。
5. 详细的前端开发文档请查看 [`frontend/README.md`](./frontend/README.md)
